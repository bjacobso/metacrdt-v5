onlang

A typed Lisp grounded in time.

pre-alpha · research-stage

onlang inherits from Clojure — immutable data, macros, REPL, Datalog, time travel — and adds what Clojure users have been patching in for a decade: a real type system, a capability-based effect system, and a time-traveling triple store that is the database and the elaboration context.

What onlang is

  • A Lisp. Homoiconic, macro-first, REPL-driven. If you've written Clojure, the surface will feel familiar.
  • Typed. Bidirectional elaboration with typed macros. Macros produce typed IR, not just syntax. Errors point at the source form, not the expansion.
  • Effectful. !-suffixed operations resolve through capabilities in scope. No IO monad, no row polymorphism at the surface — handlers are resolved at elaboration time.
  • Grounded in time. The substrate is a time-traveling triple store with Datalog queries. Ontologies, schemas, types, and deployments are all first-class ontology entities.

A taste

(define-schema Blob
  {:id      Uuid
   :content Bytes
   :mime    (Refine String :mime-type)})

(define-error BlobNotFound {:id Uuid})

(define-api-group blob-api
  (endpoint :upload
    :method :post :path "/blobs"
    :request  {:content Bytes :mime String}
    :response Blob
    :errors   [BlobNotFound])

  (endpoint :get
    :method :get :path "/blobs/:id"
    :params {:id Uuid}
    :response Blob
    :errors   [BlobNotFound]))

One source feeds the type checker, the HTTP router, a codegen target (TypeScript + Effect first; Rust and Go follow), and the ontology's own schema registry.

Why

Clojure taught us: data beats functions, functions beat macros, simple beats easy. onlang keeps that. But Clojure's users have been bolting on types, effects, and external graph stores for ten years. onlang's bet is that these aren't separate tools. They're one primitive. The type system, the effect system, and the runtime store are the same time-traveling ontology.

Prior art

onlang stands on the shoulders of Clojure (data and macros), Racket/Turnstile+ (typed macros), Lean 4 (elaborator reflection), Effekt and Scala's capability calculus (effect handlers), and Datomic/XTDB (time-traveling triples). None of the individual ideas are new. The bet is that the combination — a Lisp whose elaborator runs over a time-traveling ontology — is.

Status

Pre-alpha. The engine compiles and runs native, JS, and Wasm. The HTTP DSL has a landing slice. The effect system is a spike. There is no stable API, no release, no published package. Reading is welcome; production is not.
